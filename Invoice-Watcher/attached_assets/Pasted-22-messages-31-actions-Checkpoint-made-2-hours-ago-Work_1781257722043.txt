How the payment flow works:

5. Admins can cancel paid orders — Stars are automatically refunded via refundStarPayment

🖥️ Self-hosting on your own server
Requirements: Node.js 20+, pnpm, PostgreSQL, a domain with HTTPS (Telegram webhooks require it)
1. Install dependencies
pnpm install

2. Set environment variables
Create a .env or export these in your shell:
DATABASE_URL=postgresql://user:pass@localhost:5432/yourdb
JWT_SECRET=some-long-random-string
SESSION_SECRET=another-long-random-string
TELEGRAM_BOT_TOKEN=your-bot-token-from-botfather
TELEGRAM_BOT_USERNAME=yourbotname
VITE_TELEGRAM_BOT_USERNAME=yourbotname
WEBSITE_URL=https://yourdomain.com
NODE_ENV=production

3. Push the database schema
pnpm --filter @workspace/db run push

4. Build the frontend
PORT=3000 BASE_PATH=/ pnpm --filter @workspace/watch-configurator run build

Static files go to artifacts/watch-configurator/dist/public/
5. Start the API server
PORT=8080 pnpm --filter @workspace/api-server run dev

On first start it auto-registers the Telegram webhook using WEBSITE_URL.
6. Set up a reverse proxy (nginx example)
server {
listen 443 ssl;
server_name yourdomain.com;

# API (backend)
location /api/ {
proxy_pass http://localhost:8080;
}

# Frontend (static files)
location / {
root /path/to/artifacts/watch-configurator/dist/public;
try_files $uri $uri/ /index.html;
}
}

7. Verify the webhook
After starting, check it registered:
curl https://api.telegram.org/bot<TOKEN>/getWebhookInfo
